Your coding agent reads your files.
The raw bytes go to the model provider.

When Claude Code, Codex or Cursor runs Read .env, the file content is sent verbatim to the model's servers. Secrets included. shhh is a hook that redacts secrets into typed placeholders before that tool output reaches the model — the agent keeps reasoning, the raw secret never leaves your machine.

Proof, not a promise

Below is a real Claude Code session transcript — the .jsonl file Claude Code writes to disk, recording exactly what the model received. Same fake repo, same prompt, run twice: once without shhh, once with. This is the literal Read .env tool result.

shhh OFF what the model received 
loading transcript…
shhh ON what the model received 
loading transcript…

Highlighted lines differ between the two runs. The redaction is deterministic — it happens before the model is involved, so the model's own non-determinism never affects the result. The placeholder keeps the secret's type and a short hash, so the agent can still reason about “the Stripe key” without ever seeing it.

How it works

  1. 1 shhh installs as a PreToolUse hook on Read and Bash.
  2. 2 Before the tool output is handed to the model, shhh scans it for secrets — known provider patterns and high-entropy strings.
  3. 3 Each secret is swapped for a typed placeholder like [STRIPE_LIVE_KEY:sk_live_…:06b0436e]. The model gets the shape, never the value.

Install

$ git clone https://github.com/Musubi42/shhh
$ cd shhh && go build -o bin/shhh ./cmd/shhh
$ ./bin/shhh install claude-code

That wires the hook into ~/.claude/settings.json. Open claude and the next .env you read comes back redacted.
